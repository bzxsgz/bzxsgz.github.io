ESP8266接入物联网套件

Posted on 2018-07-04 | In C , ESP8266

环境搭建

可直接参照esp8266-aliyun-demo进行环境搭建

  1. 建议使用cygwin方式搭建环境,在下载链接下载包含有编译器的cygwin
  2. 获取SDK
    $ git clone --recursive https://github.com/espressif/esp8266-aliyun.git
    建议在git bash,或者cygwin环境下获取，可以避免出现\r\n的问题

编译

打开aliyun_config.h，将设备的

  • PRODUCT_KEY
  • DEVICE_NAME
  • DEVICE_SECRET
  • WIFI_SSID
  • WIFI_PASSWORD

填入，然后执行./genmisc.sh即可。编译完成后即可得到esp8266-aliyun/bin/upgrade/user1.2048.new.5.bin文件。

如果所烧录模块与默认不同，可以自行编辑gen_misc.sh文件，修改默认参数

进行过一次编译之后，如果不需要修改aliyun内容，为提高编译速度可以将gen_misc.sh文件中的

make aliyun  
make clean

注释掉

修改内容记录

  • SDK基于ESP8266官方RTOS SDK，结构类似。
  • 可以使用IOT_CloseLog()来关闭LOG打印
  • 需要轮询调用IOT_MQTT_Yield(pclient, 200)来获取下发的数据
  • 数据上传使用IOT_MQTT_Publish(void *handle, const char *topic_name, iotx_mqtt_topic_info_pt topic_msg)函数
  • 订阅Topic使用IOT_MQTT_Subscribe``函数,其中topic_handle_func``参数为接收数据时进入的回调。
  • 如果Topic配置为写入/读取，则发送时会即使返回消息，可以用来测试
  • 如果消息一直不通过轮询来接收，可能会堵塞，表现为一直报错消息句柄不正确

ESP8266接入飞燕平台

Posted on 2018-07-04 | In C , ESP8266

环境搭建

参照ESP8266在Alios-Things上的入门开发指南 （一）开发环境搭建及HelloWorld固件进行环境搭建

  1. 系统需求：windows10，安装Docker，VSCode(需包括插件alios-studio、c/c++)。
  2. 获取作者制作好的Docker镜像
    docker pull jacklan/alios-things-docker
    也可以自己搭建，请参照原文档
  3. 克隆alios-things仓库代码
    git clone https://github.com/alibaba/AliOS-Things

  4. 用VSCode打开AliOS-Things代码仓库，进入终端，执行下列命令即可进入docker环境
    docker run -v %cd%:/home/alios/AliOS-Things --name alios-things-build -it --rm jacklan/alios-things-docker
    编译时使用以下指令
    aos make helloworld@esp826
    如果后续需要再次进入docker环境可以依次执行以下指令

    建议保存为.bat文件方便使用

  5. 然后烧录即可
文件名 烧录地址
boot_v1.7_921600.bin 0x0
esp_init_data_default.bin 0x3fc000
blank.bin 0x3fe000
helloworld@esp8266-0x1000.bin 0x1000 
其中，“boot_v1.7_921600.bin" "esp_init_data_default.bin" "blank.bin"这三个文件可以在“xxx\AliOS-Things\platform\mcu\esp8266\bsp”目录找到  
“helloworld@esp8266-0x1000.bin”可以在“xxx\\AliOS-Things\out\helloworld@esp8266”找到

设备接入

注册设备

首先在https://living.aliyun.com/建立设备注意前期注册ICA标准格式，如果透传格式需在设备调试页面编辑脚本，记录下

  • PRODUCT_KEY
  • PRODUCT_SECRET
  • DEVICE_NAME
  • DEVICE_SECRET

修改到framework/protocol/linkkit/iotkit/sdk-encap/imports/iot_import_product.h文件中。

配置TSL

在设备页点击查看JSON,然后通过压缩转义网站进行压缩并转义操作，然后填入到\example\linkkitapp\linkkit_app.c中

数据传输

数据接收在 thing_prop_changed函数中
数据发送使用函数 linkkit_data_publish函数即可
